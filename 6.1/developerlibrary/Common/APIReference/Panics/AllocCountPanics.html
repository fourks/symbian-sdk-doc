Category: ALLOC COUNT

These panics are only raised in debug builds and are caused when the heap debugging functions detect a memory leak.

ALLOC COUNT 1 Expected aaaa Allocated bbbb Ln: ccc ddddd

This panic is raised by RHeap::__DbgMarkCheck() and User::__DbgMarkCheck() when the current number of allocated heap cells is not as expected.

aaa is the expected number of heap cells.

bbb is the allocated number of heap cells.

ccc is a line number; typically the source statement line containing the call to __DbgMarkCheck().

ddddd is a file name; typically the file containing the source statement containing the call to __DbgMarkCheck().
