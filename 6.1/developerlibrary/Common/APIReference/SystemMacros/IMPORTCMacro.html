IMPORT_C

#define IMPORT_C __declspec(dllexport)

Support

Supported from 5.0

Description

Import function from DLL

This should precede the declaration of a DLL function which is to be imported into an application program. It informs the compiler that the function is to be found in a DLL.

In a statically loaded DLL, an import library is needed to resolve the reference at link time.

Note that under WINS, IMPORT_C and EXPORT_C are both defined as __declspec(export). This is necessary, as a header file declaring an abstract class will be included in both base and user projects. The linker is able to infer that the user project is importing functions which are not available internally.
