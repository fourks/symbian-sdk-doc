The IrDA Sockets API

The IrDA Sockets API uses the generic service functionality provided by the Socket Server.

In order to access IrDA services through this API, it is first necessary to create and connect to an instance of an RSocketServ. This instance is used to load an appropriate set of protocols. EPOC IrDA permits two kinds of protocol sets to be loaded; they are identified by the names:

  • “IrMUX”.

  • “IrTinyTP”

The code for these two protocol sets is contained within a single dynamic link library, irda.prt.
