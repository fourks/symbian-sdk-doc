Hidden records

When a client inserts or updates a record using the CCommsDbTableView::PutRecordChanges(), it can mark the record as hidden. By default, a client cannot see hidden records: i.e. hidden records are excluded from views on tables unless:

  • an explicit request to include hidden records has been made by calling the CCommsDatabaseBase::ShowHiddenRecords()

  • the view has been built using a client constructed SQL query that explicitly includes hidden records (see CCommsDatabaseBase::OpenViewLC()).

Note

Connection preference table records cannot be marked as hidden.
