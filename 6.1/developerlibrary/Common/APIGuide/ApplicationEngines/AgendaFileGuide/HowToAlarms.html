How to set the model's alarms

All types of entry can have an alarm. The next outstanding alarm (or next few outstanding alarms) in an agenda file must be added to the alarm server's alarm queue in order to be handled by the alarm server. The agenda file API provides a class CAgnAlarm to do this.

To add an entries alarm to the alarm server's alarm queue:

  • create a CAgnAlarm object

  • call CAgnEntryModel::RegisterAlarm() to set the model's registered alarm

  • then whenever an alarmed entry is added to the file, the model calls FindAndQueueNextAlarmL() or FindAndQueueNextFewAlarmsL() to finds the agenda model's next/next few due alarm/alarms and queue them with the alarm server.
