How to insert graphics into a rtf source file

Graphics can be inserted into topic body text during CS Help conversion. The CS Help compiler uses references in Graphic Link style to determine which images to load—pictures must be stored in the .mbm multiple bitmap file format, generated from Windows bitmaps using the Bitmap Converter tool.

The process is:

  1. Move the cursor to the blank line in which the graphic is to be inserted.

  2. Select Insert Graphic from the EPOC menu. This automatically inserts the fields for a new graphic—if the file is based on the cshelp.dot template.

  3. Replace the archive name “PICTURES” with the name the folder containing your project’s mbm pictures. If you are using the suggested project structure then your graphics are already in the \Pictures directory, and there is no need to change the archive name.

  4. Replace the name field “GRAPHIC” with the name of your multibitmap file. It is important that your mbm file contain only a single bitmap.

Note

Help authors need not use the Insert Graphic menu option. The graphic link text can be manually inserted and style marked using the Graphic Link style. ARCHIVE=archive-name NAME=graphic-name
