Application information file compiler: aiftool.exe

Purpose

Aiftool creates an application information file, with extension aif, using information contained in a resource file and a multi-bitmap icon file.

Changes since v5

Note the following changes since v5:

  • .aif files are now built with Unicode resources

  • the EPOCROOT variable must be set to locate the epoc32 directory

Command line syntax

aiftool resource-file icon-file

Arguments

resource-file

A file that specifies the captions for all supported languages, the application capabilities, and the number of icons to be included in the aif file.

icon-file

A Symbian OS multi-bitmap file that contains the icons and icon masks for displaying the application at different sizes.

Notes

  • The command line syntax and brief notes may be obtained by typing aiftool on the command line.

  • The resource-file extension rss is added by Aiftool, and must not be specified.

  • There is no standard filename extension for a Symbian OS multi-bitmap file so the icon-file extension must be specified. Usually, it is specified as mbm.
